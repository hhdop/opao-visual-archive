# 画画的 o 泡 - 发布说明

## 本地预览

双击 `START_SITE.bat`，保持弹出的窗口不要关闭，再打开 `http://127.0.0.1:5174/`。

电脑重启、窗口关闭或服务异常时，需要重新双击一次。网站正式上线后，访客不需要做这一步。

## 用 GitHub Pages 对外发布

项目已经配好自动发布。首次发布时：

1. 在 GitHub 创建一个新的仓库，并把这个项目上传到仓库的 `main` 分支。
2. 打开 GitHub 仓库的 `Settings` -> `Pages`，将发布来源选择为 `GitHub Actions`。
3. 上传完成后，GitHub 会自动构建并发布。之后每次把作品更新推送到 `main`，网站都会自动更新。
4. 首次发布的站点地址通常是 `https://你的用户名.github.io/仓库名/`。

## 绑定自定义域名

域名购买后，在 GitHub 仓库的 `Settings` -> `Pages` 中填写你的域名，并按页面提示到域名服务商处添加 DNS 记录。DNS 生效后，在同一页面开启 HTTPS。

暂时不要在项目里写任何域名占位内容，等你拿到真实域名后再填。

## 更新作品

每张作品维护以下内容：作品名称、人物分类、创作年份、预览图、图片尺寸、百度网盘链接、夸克网盘链接。

预览图放在 `public/works`。建议使用 WebP，单张尽量控制在 500KB 以内。高清原图不要放进公开目录，继续通过网盘链接领取。

每次构建前都会自动检查：预览图是否存在、图片尺寸与比例是否一致、作品编号与排序是否重复，以及网盘链接是否安全。检查不通过时，网站不会被发布。

项目也包含自定义 `404.html`。旧链接或输入错误地址时，访客会看到返回作品库的页面，而不是托管平台的默认报错页。

## 现有安全设置

网站是纯静态作品库，不收集访客账号、支付或表单数据。网盘入口仅接受 `http` 和 `https` 地址；发布目录不包含旧的大尺寸原图。网页自身限制为只执行本站脚本和加载本站资源，并禁止表单提交与嵌入对象。

GitHub Pages 启用后，请在仓库 `Settings` -> `Pages` 中开启 `Enforce HTTPS`。不要在作品数据里填入任何账号、密码、支付信息或私密原图地址。
